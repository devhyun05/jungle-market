# 사용자 흐름 정의서

## 1. 문서 목적

이 문서는 크래프톤 정글 내부 중고거래 웹서비스의 MVP 사용자 흐름을 정의하기 위해 작성한다.

본 문서는 다음 작업의 기준으로 사용한다.

- 화면 목록 도출
- 와이어프레임 설계
- API 명세 작성
- ERD 설계
- 테스트 시나리오 작성
- GitHub Projects 태스크 분리

## 2. 문서 기준

본 문서는 다음 문서를 기준으로 작성한다.

- problem-definition.md
- requirements.md
- mvp-scope.md

## 3. 사용자 유형

| 사용자 유형     | 설명                                         |
| --------------- | -------------------------------------------- |
| 비로그인 사용자 | 아직 로그인하지 않은 사용자                  |
| 일반 사용자     | 로그인한 정글 내부 구성원                    |
| 판매자          | 상품을 등록하고 상품 상태를 관리하는 사용자  |
| 구매자          | 상품을 탐색하고 판매자에게 문의하는 사용자   |
| 관리자          | 신고와 게시글을 관리하는 운영 담당자         |
| 정지된 사용자   | 관리자에 의해 주요 기능 사용이 제한된 사용자 |

## 4. MVP 포함 흐름

본 문서는 MVP 범위에 포함된 다음 사용자 흐름을 다룬다.

- 로그인
- 상품 탐색
- 상품 상세 조회
- 상품 등록
- 상품 수정
- 상품 삭제
- 거래 상태 변경
- 상품 문의
- 상품 신고
- 관리자 신고 처리
- 관리자 상품 관리
- 마이페이지

## 5. MVP 제외 흐름

MVP에서는 다음 흐름을 다루지 않는다.

- 결제
- 에스크로
- 택배 거래
- 외부 사용자 거래
- 실시간 채팅
- 알림
- 찜하기
- 후기/평판
- 모바일 앱 전용 흐름

## 6. 전체 사용자 흐름 요약

### 일반 사용자 흐름

```text
로그인
→ 상품 목록 조회
→ 검색/필터 적용
→ 상품 상세 조회
→ 문의 작성 또는 신고
판매자 흐름
로그인
→ 상품 등록
→ 내 상품 확인
→ 문의 확인
→ 상품 상태 변경
→ 거래완료
관리자 흐름
관리자 로그인
→ 관리자 페이지 진입
→ 신고 목록 조회
→ 신고 대상 상품 확인
→ 게시글 숨김 처리
→ 신고 처리 완료
```

## 7. 로그인 흐름

### 7.1 정상 로그인
사용자
→ 로그인 화면 진입
→ 내부 사용자 인증 정보 입력
→ 로그인 요청
→ 인증 성공
→ 상품 목록 화면으로 이동
성공 조건
사용자는 정글 내부 구성원으로 확인되어야 한다.
로그인 성공 후 상품 목록 화면으로 이동해야 한다.
로그인한 사용자만 서비스 주요 기능을 사용할 수 있어야 한다.
예외 흐름
상황	처리
인증 정보가 올바르지 않음	로그인 실패 메시지를 표시한다.
내부 구성원이 아님	접근 불가 메시지를 표시한다.
정지된 사용자	서비스 주요 기능 접근을 제한한다.

### 7.2 비로그인 사용자 접근
비로그인 사용자
→ 보호된 URL 접근
→ 로그인 여부 확인
→ 비로그인 상태 확인
→ 로그인 화면으로 이동
처리 기준
비로그인 사용자는 상품 목록, 상품 상세, 상품 등록, 문의 기능에 접근할 수 없다.
비로그인 사용자가 보호된 URL에 접근하면 로그인 화면으로 이동시킨다.

## 8. 상품 탐색 흐름

### 8.1 상품 목록 조회
사용자
→ 로그인
→ 상품 목록 화면 진입
→ 상품 목록 조회
→ 최신순으로 상품 표시
화면에 표시할 정보
대표 이미지
상품 제목
가격
카테고리
거래 장소
판매 상태
등록 시간
성공 조건
사용자는 등록된 상품 목록을 볼 수 있어야 한다.
기본 정렬은 최신순으로 제공한다.
숨김 처리된 상품은 일반 사용자 목록에 표시하지 않는다.

### 8.2 키워드 검색
사용자
→ 상품 목록 화면 진입
→ 검색어 입력
→ 검색 실행
→ 검색 결과 목록 확인
처리 기준
검색 대상은 상품 제목과 설명을 기본으로 한다.
검색어가 없는 경우 전체 상품 목록을 보여준다.
검색 결과가 없는 경우 빈 상태 화면을 보여준다.

### 8.3 카테고리 필터
사용자
→ 상품 목록 화면 진입
→ 카테고리 선택
→ 선택한 카테고리의 상품 목록 확인
처리 기준
사용자는 카테고리별로 상품을 필터링할 수 있어야 한다.
카테고리 선택 해제 시 전체 상품 목록을 다시 보여준다.

### 8.4 판매중 상품만 보기
사용자
→ 상품 목록 화면 진입
→ 판매중 필터 선택
→ 판매중 상품 목록 확인
처리 기준
판매중 필터가 선택되면 판매 상태가 판매중인 상품만 표시한다.
예약중, 거래완료 상품은 필터 결과에서 제외한다.

## 9. 상품 상세 조회 흐름
사용자
→ 상품 목록에서 상품 선택
→ 상품 상세 화면 진입
→ 상품 상세 정보 확인
화면에 표시할 정보
상품 이미지
상품 제목
가격
설명
카테고리
상품 상태
판매 상태
거래 장소
판매자 정보
등록 시간
문의 버튼
신고 버튼
성공 조건
사용자는 상품 상세 정보를 볼 수 있어야 한다.
숨김 처리된 상품은 일반 사용자가 접근할 수 없어야 한다.
작성자는 본인 상품 상세 화면에서 수정, 삭제, 상태 변경을 할 수 있어야 한다.

## 10. 상품 등록 흐름
판매자
→ 상품 등록 버튼 클릭
→ 상품 등록 화면 진입
→ 상품 정보 입력
→ 이미지 업로드
→ 등록 요청
→ 등록 성공
→ 상품 상세 화면으로 이동
필수 입력값
항목	설명
제목	상품 제목
가격	판매 가격
설명	상품 설명
카테고리	상품 분류
상품 상태	새상품, 거의 새것, 사용감 있음 등
거래 장소	희망 거래 위치
이미지	1개 이상
성공 조건
로그인한 사용자만 상품을 등록할 수 있어야 한다.
상품 이미지는 1개 이상 등록되어야 한다.
등록 완료 후 상품 상세 화면으로 이동한다.
예외 흐름
상황	처리
필수 입력값 누락	누락된 항목을 안내한다.
이미지 미첨부	이미지 등록 필요 메시지를 표시한다.
허용되지 않은 이미지 형식	업로드 불가 메시지를 표시한다.
정지된 사용자	상품 등록을 제한한다.

## 11. 상품 수정 흐름
작성자
→ 본인 상품 상세 화면 진입
→ 수정 버튼 클릭
→ 상품 수정 화면 진입
→ 상품 정보 수정
→ 수정 요청
→ 수정 성공
→ 상품 상세 화면으로 이동
처리 기준
상품 작성자와 관리자만 상품을 수정할 수 있다.
일반 사용자는 타인의 상품을 수정할 수 없다.
기존 상품 정보는 수정 화면에 미리 입력되어 있어야 한다.

## 12. 상품 삭제 흐름
작성자
→ 본인 상품 상세 화면 진입
→ 삭제 버튼 클릭
→ 삭제 확인
→ 삭제 요청
→ 삭제 성공
→ 상품 목록 또는 마이페이지로 이동
처리 기준
상품 작성자와 관리자만 상품을 삭제할 수 있다.
삭제 전 사용자에게 확인 절차를 제공한다.
삭제된 상품은 일반 사용자 목록에 표시하지 않는다.

## 13. 거래 상태 변경 흐름
판매자
→ 본인 상품 상세 화면 진입
→ 상태 변경 버튼 클릭
→ 판매중/예약중/거래완료 중 상태 선택
→ 상태 변경 요청
→ 변경된 상태 표시
상태값
상태	의미
판매중	아직 거래 가능한 상품
예약중	특정 구매자와 거래가 진행 중인 상품
거래완료	거래가 완료된 상품
처리 기준
상품 작성자와 관리자만 거래 상태를 변경할 수 있다.
거래완료 상품도 상세 화면에서는 상태를 확인할 수 있어야 한다.
판매중 필터에서는 예약중, 거래완료 상품을 제외한다.

## 14. 문의 흐름

### 14.1 구매자 문의 작성
구매자
→ 상품 상세 화면 진입
→ 문의 버튼 클릭
→ 문의 화면 진입
→ 메시지 작성
→ 문의 등록
→ 문의 내역 표시
처리 기준
로그인한 사용자만 문의할 수 있다.
문의는 상품 단위로 관리한다.
구매자는 상품 상세 화면에서 판매자에게 문의할 수 있어야 한다.
정지된 사용자는 문의를 작성할 수 없다.

### 14.2 판매자 문의 확인 및 답변
판매자
→ 마이페이지 또는 문의 화면 진입
→ 내 상품에 대한 문의 목록 확인
→ 문의 상세 확인
→ 답변 작성
→ 답변 등록
처리 기준
판매자는 자신이 등록한 상품에 대한 문의를 확인할 수 있어야 한다.
판매자는 문의에 답변할 수 있어야 한다.
문의 참여자와 관리자 외에는 문의 내용을 볼 수 없어야 한다.

## 15. 신고 흐름

### 15.1 상품 신고
사용자
→ 상품 상세 화면 진입
→ 신고 버튼 클릭
→ 신고 사유 선택 또는 입력
→ 신고 제출
→ 신고 완료 메시지 확인
신고 사유 예시
금지 물품
허위 매물
부적절한 내용
중복 게시글
기타
처리 기준
로그인한 사용자만 신고할 수 있다.
사용자는 부적절한 상품 게시글을 신고할 수 있어야 한다.
동일 사용자는 동일 상품을 중복 신고할 수 없도록 한다.
신고된 상품은 관리자 신고 목록에 표시된다.

## 16. 관리자 신고 처리 흐름
관리자
→ 관리자 페이지 진입
→ 신고 목록 조회
→ 신고 대상 상품 확인
→ 처리 방식 선택
→ 게시글 숨김 처리
→ 신고 처리 상태 변경
처리 상태
상태	의미
대기	아직 처리되지 않은 신고
처리중	관리자가 확인 중인 신고
완료	조치가 완료된 신고
반려	신고가 유효하지 않다고 판단된 신고
처리 기준
관리자만 신고 목록을 조회할 수 있다.
관리자만 게시글을 숨김 처리할 수 있다.
숨김 처리된 상품은 일반 사용자 목록과 상세 화면에서 접근할 수 없어야 한다.
관리자는 신고 처리 상태를 변경할 수 있어야 한다.

## 17. 관리자 상품 관리 흐름
관리자
→ 관리자 페이지 진입
→ 전체 상품 목록 조회
→ 상품 상세 확인
→ 필요 시 상품 숨김 처리
처리 기준
관리자는 전체 상품 목록을 확인할 수 있어야 한다.
관리자는 숨김 처리된 상품을 구분해서 볼 수 있어야 한다.
일반 사용자는 관리자 상품 관리 화면에 접근할 수 없다.

## 18. 마이페이지 흐름
사용자
→ 마이페이지 진입
→ 내가 등록한 상품 확인
→ 내 문의 내역 확인
표시 정보
내가 등록한 상품
상품별 판매 상태
내가 참여한 문의 내역
처리 기준
로그인한 사용자만 마이페이지에 접근할 수 있다.
사용자는 본인이 등록한 상품 목록을 볼 수 있어야 한다.
사용자는 본인이 참여한 문의 내역을 볼 수 있어야 한다.

## 19. 주요 예외 흐름
예외 상황	처리
비로그인 사용자가 보호된 페이지 접근	로그인 화면으로 이동
일반 사용자가 관리자 페이지 접근	접근 불가 화면 표시
일반 사용자가 타인 상품 수정/삭제 시도	접근 불가 또는 권한 없음 메시지 표시
정지된 사용자가 상품 등록 시도	기능 제한 메시지 표시
정지된 사용자가 문의 작성 시도	기능 제한 메시지 표시
숨김 처리된 상품에 일반 사용자가 접근	상품을 찾을 수 없음 또는 접근 불가 표시
삭제된 상품에 접근	상품을 찾을 수 없음 표시
검색 결과 없음	빈 상태 화면 표시
이미지 업로드 실패	재시도 안내 표시
신고 중복 제출	이미 신고한 게시글이라는 메시지 표시

## 20. 화면 연결 요약
시작 화면	사용자 행동	이동 화면
