# 요구사항 정의서

## 1. 문서 목적

이 문서는 크래프톤 정글 내부 중고거래 웹서비스의 기능 요구사항과 비기능 요구사항을 정의한다.

본 문서는 다음 작업의 기준으로 사용한다.

- MVP 범위 결정
- 화면 설계
- ERD 설계
- API 명세 작성
- 테스트 시나리오 작성
- GitHub Projects 태스크 분리

## 2. 서비스 개요

크래프톤 정글 내부 구성원이 안전하고 편리하게 중고 물품을 사고팔거나 나눔할 수 있는 내부 전용 중고거래 웹서비스를 만든다.

서비스는 내부 사용자 인증, 상품 등록, 상품 탐색, 문의, 거래 상태 관리, 신고 및 관리자 기능을 제공한다.

## 3. 대상 사용자

| 사용자 유형 | 설명                                                |
| ----------- | --------------------------------------------------- |
| 일반 사용자 | 상품을 등록하거나 구매 문의를 하는 정글 내부 구성원 |
| 판매자      | 상품을 등록하고 거래 상태를 관리하는 사용자         |
| 구매자      | 상품을 탐색하고 판매자에게 문의하는 사용자          |
| 관리자      | 게시글, 신고, 사용자 상태를 관리하는 운영 담당자    |

## 4. 요구사항 우선순위 기준

| 우선순위 | 의미                                           |
| -------- | ---------------------------------------------- |
| Must     | MVP에 반드시 포함해야 하는 기능                |
| Should   | 가능하면 포함하되 일정에 따라 조정 가능한 기능 |
| Could    | 후순위 개선 기능                               |
| Won't    | MVP에서는 제외하는 기능                        |

## 7. MVP 포함 범위

MVP에서는 다음 기능을 포함한다.

- 내부 사용자 로그인
- 상품 등록, 수정, 삭제
- 상품 이미지 업로드
- 상품 목록 조회
- 상품 상세 조회
- 키워드 검색
- 카테고리 필터
- 판매 상태 관리
- 상품 문의
- 게시글 신고
- 관리자 신고 목록 조회
- 관리자 게시글 숨김 처리

## 8. MVP 제외 범위

MVP에서는 다음 기능을 제외한다.

- 결제
- 에스크로
- 택배 거래
- 외부 사용자 거래
- 실시간 채팅 고도화
- AI 추천
- 가격 시세 분석
- 복잡한 평판 점수
- 다국어 지원
- 모바일 앱

## 12. 확인이 필요한 사항

다음 항목은 개발 전 추가 논의가 필요하다.

- 내부 사용자 인증 방식
- 상품 문의를 댓글형으로 할지 채팅형으로 할지
- 이미지 저장 위치
- 관리자 권한을 누가 가질지

- 금지 물품 기준
- 거래완료 후 게시글 보관 기간
- 탈퇴 사용자의 게시글과 문의 내역 처리 방식
