# 문제 정의서

## 1. 문서 목적

이 문서는 크래프톤 정글 내부 중고거래 웹서비스의 문제 정의를 명확히 하기 위해 작성한다.

본 문서는 다음 의사결정의 기준으로 사용한다.

- MVP 기능 범위 결정
- 사용자 흐름 설계
- 화면 설계
- 우선순위 판단
- 운영 정책 수립

## 2. 대상 사용자

### 주요 사용자

- 크래프톤 정글 교육생
- 크래프톤 정글 수료생
- 크래프톤 정글 운영진

### 사용자 특성

- 제한된 기간 동안 캠퍼스 또는 교육 공간에서 생활한다.
- 노트북, 모니터, 키보드, 책, 생활용품 등 단기적으로 필요한 물품 수요가 있다.
- 기존 커뮤니케이션 채널을 통해 정보를 주고받는 데 익숙하다.
- 내부 구성원 간 거래이므로 외부 중고거래보다 신뢰 기대치가 높다.

## 3. 현재 상황

현재 정글 내부 구성원 간 중고거래 또는 나눔은 다음과 같은 방식으로 이루어질 가능성이 높다.

- 슬랙/디스코드/카카오톡 등 메신저 채널
- 개인 DM
- 구글 시트 또는 노션
- 오프라인 구두 전달

이 방식들은 빠르게 글을 올리기에는 좋지만, 중고거래에 필요한 정보 구조와 상태 관리를 제공하지 않는다.

## 4. 핵심 문제

크래프톤 정글 내부 구성원은 필요한 물건을 사고팔거나 나눔하고 싶을 때, 거래 정보가 여러 채널에 흩어져 있어 원하는 물건을 찾기 어렵고, 판매 상태와 거래 진행 상황을 확인하기 어렵다.

또한 운영진 입장에서는 부적절한 게시글, 허위 거래, 분쟁, 신고를 체계적으로 관리하기 어렵다.

## 5. 문제의 원인

- 거래글이 메신저 채널에 올라오면 시간이 지나면서 쉽게 묻힌다.
- 카테고리, 가격, 상품 상태, 판매 상태 기준으로 검색하거나 필터링하기 어렵다.
- 판매중, 예약중, 거래완료 상태가 명확히 관리되지 않는다.
- 판매자와 구매자 간 문의 이력이 흩어진다.
- 내부 사용자만 거래하는 구조를 보장하기 어렵다.
- 신고, 제재, 게시글 숨김 등 운영 도구가 부족하다.

## 6. 문제로 인한 영향

### 구매자

- 원하는 물건을 찾기 위해 여러 채널을 반복해서 확인해야 한다.
- 이미 판매된 상품인지 알기 어렵다.
- 상품 상태, 가격, 거래 장소 정보를 비교하기 어렵다.

### 판매자

- 거래 가능 여부를 반복해서 안내해야 한다.
- 문의가 여러 채널로 흩어져 관리가 어렵다.
- 판매 완료 후에도 추가 문의를 받을 수 있다.

### 운영진

- 금지 물품이나 부적절한 게시글을 빠르게 파악하기 어렵다.
- 신고 내역을 추적하거나 처리 상태를 관리하기 어렵다.
- 사용자 간 분쟁이 발생했을 때 참고할 수 있는 기록이 부족하다.

## 7. 해결 방향

정글 내부 사용자만 접근할 수 있는 중고거래 웹서비스를 제공한다.

서비스는 다음 기능을 통해 문제를 해결한다.

- 내부 사용자 인증
- 상품 등록 및 수정
- 이미지 기반 상품 상세 정보 제공
- 카테고리/가격/상태 기반 검색 및 필터
- 판매중/예약중/거래완료 상태 관리
- 상품별 문의 또는 채팅
- 찜하기
- 신고 기능
- 관리자용 게시글/신고 관리

## 8. 문제 정의문

크래프톤 정글 내부 구성원은 캠퍼스 생활 중 필요한 물건을 사고팔거나 나눔하고 싶을 때, 슬랙/디스코드/카카오톡 등 흩어진 채널을 통해 거래해야 하는 불편을 겪고 있다.

이 문제의 원인은 기존 채널이 중고거래에 특화되어 있지 않아 상품 검색, 카테고리 분류, 판매 상태 관리, 거래 이력 관리가 어렵기 때문이다.

그 결과 사용자는 원하는 물건을 찾기 어렵고, 판매자는 거래 완료 여부를 반복해서 안내해야 하며, 운영진은 부적절한 거래나 신고를 체계적으로 관리하기 어렵다.

따라서 우리는 내부 인증 사용자만 이용할 수 있는 중고거래 웹서비스를 제공하여, 상품 등록, 검색/필터, 거래 상태 관리, 문의, 신고/관리 기능을 통해 정글 내부 중고거래를 더 쉽고 안전하게 만든다.

## 9. 이번 프로젝트에서 해결할 문제

MVP에서는 다음 문제를 우선 해결한다.

- 내부 사용자가 상품을 쉽게 등록할 수 있어야 한다.
- 사용자가 판매중인 상품을 쉽게 찾을 수 있어야 한다.
- 구매자가 판매자에게 문의할 수 있어야 한다.
- 판매자가 상품 상태를 판매중/예약중/거래완료로 관리할 수 있어야 한다.
- 사용자가 부적절한 게시글을 신고할 수 있어야 한다.
- 관리자가 신고된 게시글을 확인하고 조치할 수 있어야 한다.

## 10. 이번 프로젝트에서 해결하지 않을 문제

MVP에서는 다음 문제를 우선 해결하지 않는다.

- 결제/에스크로
- 택배 거래
- 외부 사용자 대상 거래
- 자동 추천 알고리즘
- 복잡한 평판 점수 시스템
- 가격 시세 분석
- AI 상품 설명 생성
- 다국어 지원

## 11. 성공 기준

MVP의 성공 기준은 다음과 같다.

- 내부 사용자가 상품을 등록하고 조회할 수 있다.
- 사용자가 검색 또는 필터를 통해 원하는 상품을 찾을 수 있다.
- 상품의 판매 상태가 명확히 표시된다.
- 구매자가 판매자에게 문의할 수 있다.
- 부적절한 게시글을 신고할 수 있다.
- 관리자가 신고 내역을 확인하고 게시글을 숨김 처리할 수 있다.

## 12. 핵심 지표

서비스 출시 후 다음 지표를 추적한다.

- 가입 사용자 수
- 주간 활성 사용자 수
- 등록 상품 수
- 상품 상세 조회 수
- 문의 발생 수
- 거래완료 처리 수
- 신고 발생 수
- 신고 처리 완료 수

## 13. 의사결정 원칙

기능 추가 여부는 다음 기준으로 판단한다.

1. 정글 내부 중고거래의 탐색, 신뢰, 상태 관리 문제를 직접 해결하는가?
2. MVP 기간 안에 구현 가능한가?
3. 운영진이 관리 가능한 수준의 복잡도인가?
4. 사용자에게 새로운 부담을 만들지 않는가?
5. 외부 중고거래 서비스와 경쟁하기보다 내부 커뮤니티 거래에 집중하는가?
